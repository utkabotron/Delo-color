# Delo-Color — ТЗ на UI/UX дизайн

## 1. О продукте

**Delo-Color** — внутренний веб-инструмент для пакетной перекраски продуктовых фотографий мебели (delo-design.com) в стандартные цвета RAL. Позволяет загрузить фото изделия и автоматически получить 26 вариантов в разных цветах для каталога.

**Целевая аудитория:** контент-менеджер / маркетолог delo-design.com (1-3 человека).

**Платформа:** веб-приложение, desktop-first (работа с большими изображениями), mobile-friendly.

**Стек:** FastAPI + Jinja2, Alpine.js, Tailwind CSS. Без авторизации.

## 2. Дизайн-система

Визуальный стиль наследуется от **Delo-Bot** (Telegram Mini App калькулятор заказов). Ключевое направление: **iOS 26 / Apple HIG** — минимализм, крупные скругления, чистая типография.

## 4. Экраны

### 4.1. Дашборд — Список проектов (`/`)

**Layout:** заголовок + сетка карточек.

#### Шапка
- Слева: логотип/название **«Delo-Color»** (24px, bold)
- Справа: подпись «Пакетная перекраска» (14px, hint)

#### Кнопка «+ Новый проект»
- Расположение: справа под шапкой, перед сеткой
- Стиль: Primary button

#### Сетка проектов
- **Desktop:** 3 колонки
- **Tablet:** 2 колонки
- **Mobile:** 1 колонка

#### Карточка проекта
```
┌─────────────────────────────────┐
│                                 │
│  Кухонный гарнитур «Модерн»    │  ← название, semibold
│  12 фото · 26 мар 2026         │  ← caption, серый
│                                 │
│                      [Удалить]  │  ← danger text, по ховеру
│                                 │
└─────────────────────────────────┘
```

- Фон: белый, border, shadow
- Ховер: shadow усиливается
- Клик по карточке → переход на страницу проекта
- «Удалить» — текстовая кнопка, красная, с confirm-диалогом

#### Пустое состояние (нет проектов)
- Иконка: крупная (48px), нейтральная (например, папка или палитра)
- Текст: «Нет проектов» (16px, серый)
- Подтекст: «Создайте первый проект для начала работы» (14px, hint)

### 4.2. Страница проекта (`/projects/{id}`)

Одностраничный интерфейс из 5 последовательных секций. Секции появляются по мере продвижения по флоу.

#### Навигация
- `← назад` (ссылка на дашборд)
- Название проекта (24px, bold)
- Индикатор «Обработка...» (оранжевый, пульсирующий) — виден когда запущена фоновая задача


#### Секция 2: Галерея загруженных фото

**Видима когда загружено хотя бы 1 фото.**

```
  Изображения (5)                    [Создать маски]
  ┌────┐  ┌────┐  ┌────┐  ┌────┐  ┌────┐  ┌────┐
  │    │  │    │  │    │  │    │  │    │  │    │
```

- **Сетка:** 6 колонок desktop, 4 tablet, 2 mobile
- **Миниатюра:** aspect-ratio 1:1, object-fit cover, radius 12px, border

**Кнопка «Создать маски»:**
- Стиль: Success (зелёный)
- Disabled если: все фото уже с масками ИЛИ идёт обработка

#### Секция 4: Галерея результатов

**Видима когда есть хотя бы 1 результат перекраски.**

```
  Результаты (130)                           [Скачать ZIP]

  photo1.jpg:
  ┌────┐ ┌────┐ ┌────┐ ┌────┐ ┌────┐ ... ┌────┐
```

- **Группировка:** результаты сгруппированы по исходному изображению

**Кнопка «Скачать ZIP»:**
- Стиль: Neutral (тёмно-серый)
- Скачивает все результаты проекта одним архивом

## 5. Состояния и обратная связь

### 5.1. Пустые состояния

| Где | Иконка | Текст | Подтекст |
|-----|--------|-------|----------|
| Дашборд без проектов | Папка/палитра | «Нет проектов» | «Создайте первый проект» |
| Проект без фото | Изображение/камера | «Нет фото» | «Загрузите изображения для начала» |
| Нет результатов | — | Секция не отображается | — |

### 5.2. Индикаторы загрузки

- **Фоновая задача:** текст «Обработка...» рядом с заголовком, оранжевый, пульсирующий
- **Бейджи на фото:** пульсируют для `masking` и `recoloring`
- **Кнопки:** disabled во время обработки

### 5.3. Ошибки

- **Бейдж «Ошибка»** на фото: красный, без пульсации
- Визуально отличается от остальных состояний

## 7. Интерактивные паттерны

| Паттерн | Описание |
|---------|----------|
| **Drag & drop** | Зона загрузки подсвечивается при перетаскивании (синяя рамка, голубой фон) |
| **Множественная загрузка** | Можно выбрать / перетащить несколько файлов за раз |
| **Polling (опрос)** | Каждые 2 сек запрашивается статус. Бейджи обновляются без перезагрузки |
| **Динамический счётчик** | Кнопка «Перекрасить (N цветов)» обновляется при выборе |
| **Ховер на миниатюрах** | Появляется кнопка удаления (×), усиливается тень |
| **Клик на результат** | Lightbox с полноразмерным изображением |
| **Confirm перед удалением** | Нативный confirm или модалка |
